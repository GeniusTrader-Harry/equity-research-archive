About

I'm Wenhao Zhu — Economics, Finance & Data Science at Imperial College London. I've spent buy-side and IBD internships covering single names, and this archive holds the pitches I've cracked from scratch for recruiting prep and personal research.

Methodology

Every pitch on this site is produced through a 13-phase thesis-first workflow I built as a Claude Code skill. The order matters:

  • Phases 1–6 build the foundation — filings, business understanding, industry, driver decomposition, consensus map, asymmetry hunt — without picking a side.
  • Phase 7 is the direction commit (LONG / SHORT / PASS). Direction commits before any model is built. The model is the test of the view, not the source of it.
  • Phases 8–13 develop pillars, steel-man, audit, build the DCF with sensitivity and tornado, and write up the memo.

The 5-element pillar template (claim · driver · mechanism · magnitude · timeframe) is enforced. Every pillar has a pre-specified killing condition — a written-down event or number that would prove me wrong. The memo includes a "What would change my mind" section. Source code for the skill is open: GeniusTrader-Harry/equity-research.

What's on this site

Each entry has the full memo as a PDF (10–18 pages, including DCF appendix) and the underlying Excel model with assumptions, projections, and sensitivity. Working files — driver trees, pillar drafts, killing conditions — stay on disk; only the finished memo and the model are public.
